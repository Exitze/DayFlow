# Dayflow Privacy Policy

Last updated: May 3, 2026

This document is the source copy for the public Privacy Policy URL required in App Store Connect.

## English

Dayflow is a local-first iOS planner. The current version stores activities, notes, shifts, schedules, completion state, statistics inputs, and visual settings locally on your device using Apple UserDefaults.

Dayflow does not create accounts, does not require sign-in, does not send Dayflow data to a server, does not sell data, does not use advertising networks, and does not use tracking.

You can delete local app data from Dayflow Settings:

- Clear completed activities
- Clear calendar details
- Reset all local data

If Dayflow later adds sync, analytics, accounts, notifications, cloud storage, or third-party SDKs, this policy and the App Store privacy details must be updated before release.

## Русский

Dayflow - локальное iOS-приложение для планирования. Текущая версия хранит активности, заметки, смены, график, статус выполнения, данные для статистики и настройки внешнего вида локально на устройстве через Apple UserDefaults.

Dayflow не создает аккаунты, не требует входа, не отправляет данные Dayflow на сервер, не продает данные, не использует рекламные сети и не использует трекинг.

Локальные данные можно удалить в настройках Dayflow:

- Очистить выполненные активности
- Очистить детали календаря
- Сбросить все локальные данные

Если в Dayflow позже появятся синхронизация, аналитика, аккаунты, уведомления, облачное хранение или сторонние SDK, эту политику и App Store privacy details нужно будет обновить до релиза.

## App Store Connect Note

For the current local-only version, App Store Connect privacy answers should match the actual implementation: no tracking and no collected data, unless new SDKs or network features are added before release.
